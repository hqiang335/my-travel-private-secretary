# 酒店与餐饮调研

## 酒店原则

酒店从行程中心向外找：优先贴近核心景区簇、地铁、返程车站/机场通勤便利点，不只按价格排序。

## 高德酒店位置筛选

先用高德确定位置，再用飞猪查价格和预订。

- `maps_around_search`：围绕核心景区簇、地铁站、商圈搜索酒店。
- `maps_text_search`：按酒店名或商圈补查 POI。
- `maps_search_detail`：获取酒店地址、坐标、商圈。
- `maps_direction_transit_integrated`：酒店到每日首个景点、返程站点的公共交通。
- `maps_direction_driving`：酒店到景点/车站的打车时间。
- `maps_distance`：校验酒店与核心区域距离。

酒店入选前必须至少比较：到核心景区、到返程站点、到晚餐区域三类通勤。

## 酒店搜索

```bash
flyai search-hotel --dest-name "城市" --hotel-types "类型" --check-in-date "YYYY-MM-DD" --check-out-date "YYYY-MM-DD"
flyai keyword-search --query "城市 商圈 酒店 亲子/情侣/高端/经济"
```

补充小红书：

- `城市 住宿 推荐`
- `城市 商圈 住哪里`
- `酒店名 真实评价`
- `酒店名 避雷`

## 餐饮原则

餐厅围绕动线找：景区附近、酒店附近、通勤中转点附近。避免为了吃饭制造长距离折返。

## 高德餐饮位置筛选

- `maps_around_search`：围绕景区出口、酒店、商圈搜索餐厅。
- `maps_text_search`：搜索菜系或餐厅名。
- `maps_search_detail`：获取地址、坐标、商圈、类型。
- `maps_direction_walking`：景区出口到餐厅是否可步行。
- `maps_direction_transit_integrated` / `maps_direction_driving`：餐厅到下午景点或酒店。

餐厅入选前必须说明它服务哪一段动线：景区前、景区后、酒店附近、返程前。

## 餐饮搜索

小红书：

- `城市 美食 必吃`
- `景点名 附近 美食`
- `商圈名 餐厅 排队`
- `餐厅名 真实评价`
- `城市 不踩雷 餐厅`

网络或工具补图时记录来源。餐馆如涉及排队，必须写建议到达时间或替代店。
