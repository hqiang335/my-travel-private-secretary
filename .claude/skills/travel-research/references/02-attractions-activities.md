# 景点与活动调研

## 目标

找到匹配用户兴趣和节奏的核心景区、活动、节庆、体验项目，并保留门票、预约、图片和真实评价。

## 高德 POI 搜索

先用高德建立地点底图，再用小红书和飞猪补充评价与票务。

- `maps_text_search`：搜索 `城市 + 景区/博物馆/公园/商场/夜市/演出/展览`。
- `maps_search_detail`：查询候选 POI 的地址、坐标、商圈、类型。
- `maps_around_search`：围绕核心景区搜索附近景点、商场、室内备选。
- `maps_distance`：初筛景点之间距离，形成地理簇。
- `maps_direction_transit_integrated` / `maps_direction_driving`：验证簇内通勤是否合理。

高德输出用于决定“能不能排在同一天”，小红书输出用于判断“值不值得去”。

## 小红书搜索模板

小红书低频串行搜索，先查缓存；无缓存时每次只搜索 1 个关键词。

优先用窄关键词：

- `目的地 天数 攻略`
- `目的地 避坑 排队 预约`
- `景点名 真实评价`
- `景点名 排队 时间`
- `景点名 拍照 机位`
- `景点名 雨天/高温/周末`

避免一次性搜索过宽词，如只搜 `目的地 攻略`。如果必须用宽词，只搜 1 次并缓存结果。

超时后：

- 等待后改用更窄关键词重试 1 次。
- 仍失败则停止该主题小红书搜索。
- 在候选点中写“小红书评价：待补查”，继续使用高德 POI 和飞猪票务。

## flyai-cli 搜索模板

```bash
flyai search-poi --city-name "城市" --keyword "景点名 门票"
flyai keyword-search --query "城市 景点名 门票 预约"
flyai keyword-search --query "城市 当季 活动 展览 演出"
```

## 每个候选点记录

```markdown
### 名称
- 类型：
- 高德 POI/地址/坐标：
- 所属地理簇：
- 推荐理由：
- 匹配偏好：
- 建议游玩时长：
- 开放/预约：
- 门票/价格：
- 跳转链接：

- 适合放在哪一天：
```

## 筛选规则

- 按地理位置聚类，不要只按热度排序。
- 同一天景点之间必须有高德路线支撑；不能只凭地图直线距离。
- 对紧凑节奏保留 4-5 个点/日；适中 2-3 个点/日；悠闲 1-2 个点/日。
- 热门景区必须检查预约、闭馆日、排队和最佳到达时间。
- 天气敏感活动必须准备室内替代。
- 小红书缺失不能阻塞景点候选，但必须标注真实评价待补查。
