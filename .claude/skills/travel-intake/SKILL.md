# Travel Intake

## 目标

在调研和排日程前，确认足够的用户偏好和时间边界。优先从用户原话抽取信息，只问缺失项。

问卷必须优先使用选择题，降低用户手打成本。只有用户选择“其他/补充/不确定”或关键时间地点无法用选项表达时，才让用户输入文本。

## 先问硬边界

必须先确认：

- 目的地或候选目的地。
- 出发城市、出发日期、可接受出发时间段。
- 返程城市、返程日期、可接受返程时间段。
- 大交通偏好：高铁、飞机、自驾、包车、价格优先或时间优先。
- 同行人构成、行动能力、证件/签注限制。

未确认出发和返程边界时，不得进入每日行程规划。

硬边界中日期、城市、已购票班次可以用文本；偏好项尽量用选择题。

## 再问偏好

按批次提问，每批最多 4 个问题。使用 `references/questionnaire.md` 的选项模板：

- 预算：人均总预算或每日预算，经济/舒适/豪华。
- 住宿：酒店/民宿/连锁/客栈，位置权重，停车/洗衣/浴缸/健身房等硬需求。
- 市内交通：地铁公交、打车、骑行、自驾/租车。
- 饮食：辣度、餐饮风格、过敏、清真、素食。
- 节奏：悠闲、适中、紧凑；是否需要午休或拍照时间。
- 兴趣：文化历史、自然风光、主题乐园、探店、夜市、非遗、购物。
- 避雷：人挤人、强制购物、过度商业化、排队、照骗。
- 数字需求：充电宝依赖、离线地图、抢票接受度。

## 选择题执行

- 使用结构化选择题工具时，单选用于预算、节奏、交通偏好；多选用于兴趣、住宿硬需求、饮食限制、避雷。
- 每题选项保持短句，必须包含“其他/补充”或“不确定，先用默认值”。
- 用户选择“其他/补充”后，只追问对应题目的文本，不重新问整批。
- 问卷完成后把选择项转换成可执行偏好，不要只复述选项。

## 输出格式

完成后输出结构化摘要：

```markdown
## 需求摘要
- 目的地：
- 出发边界：
- 返程边界：
- 同行人：
- 预算：
- 住宿偏好：
- 交通偏好：
- 饮食偏好：
- 节奏与兴趣：
- 硬性限制：
- 避雷事项：
- 待确认：
```

## 追问规则

- 如果缺少出发/返程时间，只问交通边界，不问细节偏好。
- 如果能用选择题表达，就不要让用户手打。
- 每批选择题最多 4 个问题；多选题用于兴趣、硬需求、避雷、餐饮限制。
- 每组选择题保留“其他/补充”和“不确定，先用默认值”。
- 用户选择“其他/补充”后，再用一次文本追问收集具体内容。
- 如果用户给出模糊预算，转成可执行区间。
- 如果有老人、儿童、孕妇、宠物、行动不便，必须记录为硬约束。
- 如果用户要求“你看着安排”，默认适中节奏、舒适型预算、地铁加打车混合，并在摘要中标注默认值。
