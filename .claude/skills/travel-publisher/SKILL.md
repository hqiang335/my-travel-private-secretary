# Travel Publisher

## 目标

把调研结果转成可执行攻略，写入 Notion Database，并返回页面 URL。发布前必须按 `notion-output.md` 验收。

## 成稿原则

- 用用户偏好解释每个核心选择。
- Notion Database 必须使用画廊视图，页面必须有封面图、摘要卡片和清晰分区。
- 每日行程按时间顺序写，保留午休/通勤/返程缓冲。
- 每个酒店、票务、交通产品、小红书核心笔记都附链接。
- 选定景区、餐馆、酒店建议配图；优先使用小红书或工具返回图片，其次使用可公开访问的网络图片。
- 每段关键通勤写明方式、距离、预计时间。
- 价格全部标注“参考价格，以实时为准”。
- 对天气、安全、预约失败给出替代方案。

## Notion 写入

使用项目级 Notion MCP。若 MCP 不可用，可请求用户重载 MCP 配置；不要把密钥写入 skill。

目标 Database：

- 优先使用用户指定的 Database。
- 未指定时，查找或创建“旅游攻略库”。
- Database 至少包含：目的地、日期、天数、同行人、预算、节奏、状态、封面图、Notion URL。
- 设置或提示用户使用画廊视图；每张卡片以封面图、目的地、日期、主题摘要呈现。

## 页面结构

按 `notion-output.md` 的顺序组织页面。不要先写长文再补表格；每个模块都要有可执行信息。

## 发布后汇报

返回：

- Notion 页面 URL。
- 适合用户偏好的 3-5 个核心决策。
- 关键预订链接。
- 需要用户最终确认的事项。
- 全维度验收结果。
