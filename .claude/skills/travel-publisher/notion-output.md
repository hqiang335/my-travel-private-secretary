# Notion 输出模板

## 页面顺序

0. Database 属性：目的地、日期、天数、同行人、预算、节奏、状态、封面图。
1. 页面封面：目的地代表图；画廊卡片也使用同一封面。
2. 标题：目的地、日期、同行人、攻略版本时间。
3. 旅行摘要卡片：预算、节奏、核心兴趣、硬约束、天气提醒。
4. 出发与返程：大交通、到离站时间、两端接驳、返程缓冲。
5. 每日行程：上午、午餐、下午、晚餐、夜间/休息、住宿。
6. 通勤矩阵：机场/车站/酒店/景区/餐厅之间的方式、距离、时间。
7. 景区与活动卡片：图片、亮点、门票、预约、适配偏好、链接。
8. 酒店方案：图片、位置、设施、价格、优缺点、预订链接。
9. 餐饮清单：图片、口味、招牌菜、排队/预约、动线、链接。
10. 票务与预约：门票、预约时间、抢票提醒、跳转链接。
11. 预算表：住宿、餐饮、活动、交通、杂项。
12. 天气与备选：天气预警、室内替代、雨天动线。
13. 文化礼仪与安全：禁忌、诈骗、限行、医院/应急点。
14. 打包与时间轴：行前准备、证件、设备、保险、离线地图。
15. 小红书参考：核心笔记标题、要点、链接。
16. 发布前验收：全维度是否已落实。

## 视觉排版

- Database 使用 Gallery 视图；卡片预览使用页面封面图。
- 页面顶部用 1 张大封面图，摘要区用 callout 或表格做成信息卡片。
- 景区、餐馆、酒店采用卡片式小节：图片在前，随后是推荐理由、关键事实、链接。
- 表格只用于比较：酒店对比、预算、通勤矩阵、票务预约。
- 长段落拆成短 bullet；每个模块先给结论，再给依据。
- 同类模块命名保持一致，便于 Notion 页面扫读。

## 图片规则

- 选定景区、餐馆、酒店建议配图；核心景区和核心餐馆必须尽量配图。
- 优先级：小红书笔记图片 > 飞猪返回图片 > 公开网络图片 > 无图并标注“需补图”。
- 图片必须可公开访问；不得使用本地路径或需要登录的临时地址作为长期封面。
- 插入 Notion 前按展示用途调整尺寸：封面用横图，卡片用约 16:9 或 4:3，避免超大原图刷屏。
- 如果只能拿到小红书临时图片，允许临时嵌入，但要标注“图片链接可能失效”。

## 每日行程格式

```markdown
### Day X｜主题
- 上午：
- 通勤：
- 午餐：
- 通勤：
- 下午：
- 晚餐：
- 夜间/休息：
- 住宿：
- 当日预算：
- 备选方案：
```

## 通勤格式

```markdown
起点 → 终点｜方式｜距离｜预计时间｜备注
```

方式必须明确：地铁、公交、打车、步行、自驾或组合路线。

## 全维度验收

发布前逐项标记：

- 个人偏好建模：已落实/缺口。
- 灵感与景点探索：已落实/缺口。
- 目的地深度研究：已落实/缺口。
- 交通与住宿预订：已落实/缺口。
- 行程与节奏规划：已落实/缺口。
- 财务与预算管理：已落实/缺口。
- 文化与社会融入：已落实/缺口。
- 物资与应急时间轴：已落实/缺口。

任一项为缺口时，不得声称攻略完成；必须说明需要补查的信息。

## 链接占位规则

- 有真实链接：直接嵌入。
- 工具未返回链接：写“需二次确认”，并说明来源。
- 不得编造小红书、飞猪、Notion 链接。
- 不得编造图片来源；图片来自网络搜索时保留图片 URL 或页面来源。
