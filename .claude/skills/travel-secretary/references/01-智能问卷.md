# 第一阶段：智能问卷 & 偏好管理

## 核心原则

**智能跳过已知信息** — 从用户原始消息中提取已知信息，不再重复询问。

## 1. 读取历史档案

### 检查历史档案

## 2. 从用户消息提取已知信息

### 提取规则

| 信息类型 | 提取示例 |
|---------|---------|
| 目的地 | "想去成都" → destination: "成都" |
| 天数 | "玩3天" → days: 3 |
| 日期 | "五一假期" → travel_dates: "2026-05-01 至 2026-05-03" |
| 人数 | "两个人" → companions: "好朋友（2人）" 或 "情侣" |
| 预算 | "预算5000" → budget_per_day: 根据天数计算 |

### 提取后的处理

将已知信息标记，**不再在问卷中询问**。

- 每次最多 4 个问题

## 5. 常见问题

### Q: 用户输入"其他"选项怎么办？
A: 从 `AskUserQuestion` 的 `annotations` 字段中读取用户自定义输入。

### Q: 如何处理日期相对表达？
A: 将相对日期转换为绝对日期：
- "五一" → "2026-05-01 至 2026-05-03"
- "下周末" → 计算具体日期
- "7月" → "2026-07-01 至 2026-07-31"（待用户确认具体日期）
