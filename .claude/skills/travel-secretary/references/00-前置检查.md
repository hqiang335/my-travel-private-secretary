# 前置检查

在开始执行旅游秘书 SKILL 之前，必须进行以下前置检查。

## 1. 检查 xiaohongshu MCP 是否运行

xiaohongshu MCP 需要手动启动才能使用。

### 检查方法

```bash
# 检查进程是否运行
ps aux | grep xiaohongshu-mcp | grep -v grep
```

### 启动命令

如果未运行，需要启动 xiaohongshu MCP：

```bash
# 在后台启动 xiaohongshu MCP（macOS ARM64）
cd /path/to/xiaohongshu-mcp && ./xiaohongshu-mcp-darwin-arm64 -headless=false &
```

**重要提示**：
- `-headless=false` 参数会打开浏览器窗口用于登录验证
- 首次使用需要扫码登录小红书账号
- 启动后需要等待 5-10 秒让服务完全就绪

### 验证 MCP 可用性

启动后，可以通过以下方式验证：

```bash
# 测试 MCP 连接（通过 Claude Code 的 MCP 工具）
# 如果能成功调用 check_login_status，说明 MCP 已就绪
```

## 2. 检查其他 MCP 服务

### amap-maps MCP
- 通常随 Claude Code 自动启动
- 无需手动检查

### flyai CLI
- 通过 `npx` 调用，无需预先启动
- 首次使用会自动安装

### Notion API
- 使用 Python urllib 直接调用 REST API
- 无需 MCP 服务器
- Token 从 `~/.claude.json` 读取

## 3. 检查必要目录

确保项目目录结构存在：

```bash
# 检查并创建 travel-profiles 目录
mkdir -p /Users/q/Desktop/my-travel-private-secretary/.claude/travel-profiles/
```

## 4. 前置检查清单

在开始 SKILL 执行前，确认：

- [ ] xiaohongshu MCP 已启动并登录
- [ ] travel-profiles 目录存在
- [ ] 网络连接正常（用于调用各 API）
- [ ] Notion Token 配置正确（在 `~/.claude.json` 中）

## 5. 错误处理

如果前置检查失败：

1. **xiaohongshu MCP 启动失败**
   - 检查可执行文件路径是否正确
   - 检查是否有其他实例占用端口
   - 查看 MCP 日志输出

2. **Notion Token 无效**
   - 检查 `~/.claude.json` 中的 `OPENAPI_MCP_HEADERS`
   - 验证 Token 是否过期
   - 重新生成 Notion Integration Token

3. **网络问题**
   - 检查代理设置
   - 验证 API 端点可访问性
