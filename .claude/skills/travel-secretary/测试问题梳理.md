# 端到端测试问题梳理

## 第一阶段：智能问卷 & 偏好管理

### ✅ 运行良好
- 从用户消息中提取已知信息（目的地、天数、日期、人数）
- 分批次询问未知信息，避免重复提问
- 档案保存到 `.claude/travel-profiles/*.json`

### ⚠️ 遇到的问题
1. **AskUserQuestion filters 字段错误**
   - 错误：`MCP error -32602: unexpected additional properties ["_"]`
   - 原因：filters 对象中不能包含 `_` 字段（虽然 JSON schema 提示需要，但实际 MCP 不接受）
   - 解决：完全移除 filters 对象中的 `_` 字段

## 第二阶段：多源数据收集

### ✅ 运行良好
- xiaohongshu 搜索和详情获取功能正常
- amap-maps 天气、距离、POI 查询正常
- 并行数据收集提高效率

### ⚠️ 遇到的问题

1. **xiaohongshu MCP 需要手动启动**
   - 问题：MCP 服务器需要手动启动才能使用
   - 启动命令：`./xiaohongshu-mcp-darwin-arm64 -headless=false`
   - 需要在 SKILL 开始前检查并提示用户启动

2. **flyai CLI 参数名称必须精确**
   - `search-flight` 正确参数：
     - `--origin` (不是 `--from`)
     - `--destination` (不是 `--to`)
     - `--dep-date` (不是 `--date`)
   
   - `search-hotel` 正确参数：
     - `--dest-name` (不是 `--city`)
     - `--hotel-types` (不是 `--type`)
     - `--check-in-date` 和 `--check-out-date` (不是 `--checkin`/`--checkout`)
   
   - `search-poi` 正确参数：
     - `--city-name` 和 `--keyword` 必须分开（不能把城市名放在 keyword 里）

3. **flyai 返回数据字段问题**
   - 航班数据中 `airline` 和 `flightNo` 字段为空字符串
   - 但 `jumpUrl` (飞猪短链接) 可用
   - 需要依赖 `jumpUrl` 而非航班号

## 第四阶段：生成 Notion 攻略报告

### ✅ 运行良好
- 使用 Python urllib 直接调用 Notion API
- 分批次创建内容块（避免 100 块限制）
- Database 和 Page 创建成功

### ⚠️ 遇到的问题

1. **Notion MCP 需要重启 Claude Code**
   - 问题：添加 Notion MCP 到 `~/.claude.json` 后，需要重启 Claude Code 才能生效
   - 解决方案：使用 Python urllib 直接调用 Notion REST API
   - API Token: 从 `~/.claude.json` 的 `OPENAPI_MCP_HEADERS` 中读取

2. **Notion API 单次请求限制 100 个 block**
   - 错误：`body.children.length should be ≤ 100, instead was 112`
   - 解决：先创建页面（包含前 95 个 block），再用 PATCH 追加剩余 block

3. **需要先搜索 Database 是否存在**
   - 使用 `/v1/search` API 搜索名为 "🗺️ 旅游攻略库" 的 Database
   - 若不存在才创建新 Database

4. **Table 创建需要特殊结构**
   - Table block 的 `children` 必须是 `table_row` 类型
   - 需要在 table 对象内部定义 `table_width` 和 `has_column_header`
